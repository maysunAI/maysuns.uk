How I Work With AI: 3 Rules You Can Steal

I've been building with tools like Claude Code for months and picked up a few collaboration habits worth sharing — no project specifics, just how I work with AI itself.

1. Keep substantive answers in one file, not scattered across chat. I have the AI write every real answer into a persistent reply log and keep the terminal output to a one-line confirmation — the log stays a complete, searchable record as conversations get longer.

2. Label trust levels in code instead of relying on memory. Tags like trust: high/medium/low tell the AI "safe to change freely" versus "check with me first," so I don't repeat instructions every time.

3. Force a clear line between "actually done" and "placeholder." Progress reports must separate what's genuinely working from what's stubbed out — this habit has saved me from more than one "I thought this was finished" surprise.

None of this needs special tooling — just communication habits. Anyone working with AI day to day can copy them directly.
