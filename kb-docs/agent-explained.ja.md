# AIエージェントとは何か

## 解決する課題

普通のチャットボットは一度に一つの質問にしか答えない。あなたが尋ね、それが返答し、そこで会話は終わる。しかし現実の多くのタスクは一問一答では済まない——「このバグを直して」「このトピックを調べて要約を書いて」「これらのファイルを整理し直して」といったものには、複数のステップ、たった今起きたことに基づいて次に何をするかの判断、そして（誰か他の人がすべきことを説明するだけでなく）実際に行動する能力が必要になる。

## それが何であるか

AIエージェントとは、一連のステップを計画し、ツールを使って実際の行動（ファイルの読み書き、コマンドの実行、APIの呼び出し、ウェブの閲覧）を取り、その行動の結果を観察し、次に何をするかを決定できるモデルであり、これを人間が一つ一つのステップを承認することなく、タスクが完了するまで繰り返す。

中核となるループはたいてい次の通りだ：**観察 → 判断 → 行動 → 結果を観察 → 再び判断。** 普通のチャットボットとエージェントを分けるのは最後の部分だ。エージェントは答えを生成するだけでなく、行動を取り、何が起きたかを見て、それに応じて調整する。

## 何が「エージェント的（agentic）」たらしめるのか

すべてのAI機能がエージェントというわけではない。有用な判定基準がある。人間が行動するためのテキストを生成するだけなのか、それとも複数ステップにわたって自ら判断し行動するのか。

- **エージェント的ではない**：「この文書を要約して」——一つの入力、一つの出力、継続的な判断はない。
- **エージェント的**：「このテストスイートを通るようにして」——モデルはテストを実行し、失敗内容を読み、何を変更すべきか判断し、コードを編集し、再びテストを実行することを、完了するまで繰り返さなければならない。

## なぜ重要なのか

- **より多くを委任できるようになる。** かつては判断のたびに人間が必要だった複数ステップの、オープンエンドなタスクを任せられるようになり、人間は各ステップをマイクロマネジメントするのではなく、結果をレビューすればよくなる。
- **ツールの使用が本物にする鍵だ。** エージェントは呼び出せるツール——ファイルアクセス、コード実行、検索、ブラウザ操作——次第の能力しか持てない。推論も重要だが、本物のツールがなければそれはただの「口先」のままだ。
- **自律性にはガードレールが必要だ。** エージェントはすべてのステップを承認なく行動するため、人間の監督なしにどこまで許可するかが実際の安全性の問題になる——これが実務でどう管理されるかについては、権限/信頼レベルの慣習を参照のこと。
