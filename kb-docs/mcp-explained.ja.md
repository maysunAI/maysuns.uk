# MCP（Model Context Protocol）とは何か

## 解決する課題

AIモデルは、単に話すだけでなく、ファイルを読んだり、データベースに問い合わせたり、APIを呼び出したり、ブラウザを操作したりできるようになるほど、ますます有用になる。しかし共通の標準がなければ、モデルをツールに接続したいすべてのアプリケーションは、独自の一度限りの統合を構築しなければならなくなる——モデルとツールの組み合わせごとに違うプラグが必要になる。これはスケールしない。Nモデル×Mツールは、N×Mのカスタム統合を意味する。

## それが何であるか

MCP（Model Context Protocol）は、AIモデルを外部ツールやデータソースに共通のプロトコルを通じて接続するためのオープンな標準だ。ツールやデータソースは一つのMCP互換インターフェース（「サーバー」）を実装し、MCP互換の任意のAIアプリケーション（「クライアント」）は、その特定の組み合わせ専用の統合を作らなくても、それに接続できる。

役立つ例え話がある。**ハードウェアにおけるUSBだ。** 共通の標準がなかった頃、周辺機器はそれぞれ独自の専用コネクタを必要とした。USBが登場すると、デバイスは一度USBを話せるようにするだけで、あらゆるUSB互換ホストと動作するようになった。MCPはAIとツールの接続について同じことを目指している——統合を一度書けば、それが一つのアプリケーションだけでなく、MCP互換の任意のAIアプリケーションで動作する。

## 何を標準化しているのか

大まかに言うと、MCPは以下のための共通の方法を定義している。

- ツール／データソースが何を提供しているか（どんな操作が可能か、どんなデータが読み取れるか）を記述すること
- AIアプリケーションがそれらの能力を一貫した形式で発見し、呼び出すこと
- 構造化された結果がモデルに予測可能な形で返ってくること

## なぜ重要なのか

- **一度限りの統合が減る。** データベース、ファイルシステム、ブラウザ自動化ツール、社内APIはそれぞれ一つのMCPサーバーを公開するだけで、アプリケーションごとに独自の接着コードを必要とすることなく、あらゆるMCP互換AIクライアントから利用できるようになる。
- **合成可能性（composability）。** インターフェースが標準化されているため、AIアプリケーションは複数のMCPサーバー——ファイルアクセス、検索ツール、ブラウザ——に同時に接続し、それぞれ異なる呼び出し規約を使い分けるのではなく、統一的に扱える。
- **エコシステムの成長が速くなる。** ツールの開発者はプラットフォームごとに個別の統合作業をする必要がなくなり、一度作ればMCP互換のあらゆるAIアプリケーションに届けられるようになる。
