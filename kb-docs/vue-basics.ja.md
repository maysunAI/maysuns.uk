# Vueの基本

## それが何であるか

Vueはユーザーインターフェースを構築するためのJavaScriptフレームワークだ。中核となる考え方は**宣言的で状態駆動のレンダリング**である。DOMの変更を手作業でプッシュ・プルするコード（「このボタンがクリックされたら、この要素を見つけてテキストを変更する」）を書く代わりに、*ある状態が与えられたときに*ページがどう見えるべきかを記述し、その状態が変わるたびに実際のDOMを更新する作業はVueが引き受ける。

## 中核となる概念

- **コンポーネント（Components）** — 自己完結した再利用可能なUIの単位（ボタン、カード、ページ全体のセクションなど）で、それぞれが自身のテンプレート（マークアップ）、ロジック、スタイリングをまとめて持つ。
- **リアクティビティ（Reactivity）** — コンポーネントが依存しているデータの一部が変化すると、Vueは何を更新すべきかを手動で追跡することなく、それに依存するUIの部分を自動的に再レンダリングする。
- **テンプレート（Templates）** — コンポーネントが何をレンダリングするかを記述するための、HTMLに似た構文（いくつかのディレクティブが追加されている）で、マークアップを普通のHTMLに近い状態に保ち、純粋にJavaScriptで生成されたマークアップよりも読みやすくする。
- **単一ファイルコンポーネント（Single-file components）** — コンポーネントのテンプレート、スクリプト、スコープ付きスタイルを一つのファイルにまとめた `.vue` ファイルで、関連するコードを物理的に近い場所にまとめておける。

## 学習曲線が緩やかだと言われる理由

- Vueのテンプレートは普通のHTMLに近く見えるため、一部の代替フレームワークと比べて、最初に学ぶべき新しい構文が少ない。
- **段階的な導入（Incremental adoption）**が実際にサポートされている使い方だ。既存のページの一部分（たとえば一つのインタラクティブなウィジェットだけ）にVueを追加することができ、サイト全体をVue中心に作り直す必要はない——最初からすべてを賭ける必要がない。
- 良質な公式ドキュメントと、比較的小さな概念の中核（コンポーネント、テンプレート、リアクティブな状態）のおかげで、「動くものを作れた」までの最初の道のりが短い。

## いつ使うべきか

Vueは、ページに意味のあるインタラクティブ性——変化する状態と、それに同期し続ける必要のあるUI——が必要で、それが普通のHTML/CSS/素のJavaScriptで手軽に扱える範囲を超えるときに、良い選択肢になる。動的な振る舞いのない完全な静的ページの場合、Vueのようなフレームワークは不要なオーバーヘッドになる。その価値は、些細ではないUIの状態を管理するようになって初めて発揮される。
