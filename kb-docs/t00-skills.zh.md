# 技能（Skill）：按需加载的指令集

## 解决的问题

一个能做很多事情的AI助手——写营销文案、审查安全性、排版简历、为了测试而操控浏览器——面临一种张力：如果把这些能力各自的完整指令都「以防万一」加载进每一次对话，那么助手在大多数时间里都在背负大量无用的重量，而且指令的绝对体量本身就可能挤占对眼前实际任务的注意力。

## 核心思路

「技能」是为某个特定工作流、工具或领域打包好的、自成一体的一套指令——助手只在这套指令**确实与眼前任务相关**时才加载它，而不是默认在每次对话中都加载。一个用于「审查一个拉取请求（pull request）」的技能，在你问一个完全无关的话题时不需要被加载；只有当那类任务真正出现时，它才会被调用进来。

这在概念上类似于软件中的延迟加载（lazy loading）：你不会在启动时导入每一个库，而是在真正用到某个代码路径时才加载相应的模块。

## 为什么重要

- **让日常交互保持轻量。** 大多数对话并不需要大多数能力。只加载相关的部分能保持效率。
- **让专业化的深度得以存在，而不拖累其他一切。** 一个技能可以做到需要的详细和具体程度——长长的清单、专业术语、边界情况——而不会让这些细节拖累无关的对话。
- **随着能力增长而更好地扩展。** 增加第一百个专门的技能，并不会让当天第一次对话变慢，因为大多数技能除非需要,否则永远不会被加载。

## 如何自己应用这套方法

如果你发现自己在为AI助手维护一套越来越庞大的「如何做X」的指令集，可以考虑把它们拆分成独立的、标注清楚的包，而不是一个不断膨胀的主提示词——并且只在对应类型的任务真正出现时才加载各自的那一份。
