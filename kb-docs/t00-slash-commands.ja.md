# カスタムスラッシュコマンド

## 解決する課題

AIアシスタントへの指示の中には、一度きりではないものがある——「このコードをチェックリストと照らし合わせてレビューして」「このタスクの規模を見積もって」「このプロジェクトをリリース用にパッケージ化して」のように、何度も繰り返し使う再現可能な多段階の手順だ。同じ複数段落の指示を毎回入力するのは面倒であり、さらに悪いことに、ブレを招く——毎回微妙に違う言い回しになり、微妙に違う結果が得られてしまう。

## アイデア

スラッシュコマンドは、再現可能なワークフローを単一の再利用可能なトリガーにパッケージ化する——典型的には `/estimate` や `/publish` のような `/` で始まる短い名前だ。その一語の背後には、何をチェックするか、どの順番で行うか、どんな出力形式を作るか、何かおかしければ何を指摘するかという、一連の完全な手順一式が控えている。

一度定義してしまえば、ワークフローの呼び出しは打ち直したエッセイではなく一行のコマンドになり——同じくらい重要なことに——*一貫性*が生まれる。同じコマンドは毎回同じ種類のプロセスを生み出し、これは繰り返し頼る何か（うっかりステップを飛ばしたくない公開前チェックリストのようなもの）にとって重要だ。

## なぜ重要なのか

- **繰り返しより一貫性。** よく使うワークフローは、その日たまたま指示をどれだけうまく言い回せたかに左右されるのではなく、毎回同じように振る舞うべきだ。
- **良い習慣への摩擦が減る。** きちんとしたチェックリストを実行するのに5分の入力ではなく一語で済むなら、実際に実行する可能性が高くなる。
- **発見しやすく共有しやすい。** 名前付きコマンドのライブラリは、「いつも私がこう頼むやり方」というバラバラな記憶よりも、他の誰か（あるいは未来の自分）に渡しやすい。

## 自分なりに応用するには

だいたい同じ形で2回以上打ち込んでいることに気づいた指示を探す——それが名前付きコマンドの候補だ。手順を一度書き留め、短くて覚えやすい名前を付け、毎回ゼロから指示を組み立て直すのではなく再利用する。
