# 技能（按需加载的指令集），解释一下

## 解决的问题

一个AI智能体潜在地可以帮上几十种专门任务的忙——部署到某个特定的云服务商、遵循公司的代码审查清单、排版某种特定格式的报告。如果每一套这样的专门指令都必须「以防万一」加载进每一次对话，模型在大多数时间里就要背负大量不相关的上下文，导致响应变慢、注意力也不集中。

## 它是什么

技能是一套打包好的、专门化的指令集——描述如何完成某项具体工作——AI智能体只在它确实和眼前任务相关时才加载它，而不是让每一次对话都背负它可能用到的每一项能力的全部重量。

实际操作中，一个技能通常由两部分组成：一段简短的说明，描述*何时*该用它（这样智能体才知道要去调用它），以及具体的操作指令，说明*如何*完成这项工作（只在真正需要时才加载）。

## 它与斜杠命令有什么不同

两者相关但并不相同：斜杠命令是由人明确输入类似 `/deploy` 这样的指令来触发的。技能则通常是隐式触发的——智能体从上下文中识别出某个任务符合某个技能的描述，并自行加载它，而不需要被告知确切的触发词。

## 为什么重要

- **让日常交互保持轻量。** 大多数对话并不需要大多数技能；默认不加载它们，能让响应更快、更聚焦。
- **能扩展到大量专业知识。** 一个由数十个技能组成的库可以存在，而不会让每一个技能都拖累每一次对话——只有相关的那部分才会被调用进来。
- **鼓励可复用、有良好文档的专业知识。** 编写一个技能会迫使专业知识被清晰地写下来一次，而不是每次需要时都从头重新解释一遍。
