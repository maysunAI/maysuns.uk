# スキル：オンデマンドの指示セット

## 解決する課題

マーケティングコピーを書く、セキュリティをレビューする、履歴書をフォーマットする、テストのためにブラウザを操作するなど、多くのことができるAIアシスタントには、一つの緊張関係がある。それらの能力すべての完全な指示を「念のため」毎回の会話にロードしてしまうと、アシスタントはほとんどの時間、大量の死重を抱えることになり、指示の量そのものが目の前の実際のタスクへの注意力を奪いかねない。

## アイデア

「スキル」とは、特定のワークフロー、ツール、あるいは領域のためにパッケージ化された自己完結型の指示セットであり、アシスタントはこれを、すべての会話でデフォルトとしてではなく、目の前のタスクに実際に関連がある**ときだけ**ロードする。「プルリクエストをレビューする」ためのスキルは、まったく無関係な話題について質問している間はロードする必要がない——その種のタスクが実際に出てきたときに限って呼び出される。

これは概念的にはソフトウェアの遅延読み込み（lazy loading）に似ている。起動時にすべてのライブラリをインポートするのではなく、それを必要とするコードパスが実際に実行されたときにモジュールをロードする、というのと同じ考え方だ。

## なぜ重要なのか

- **日常のやり取りを軽く保てる。** ほとんどの会話はほとんどの能力を必要としない。関連するものだけをロードすることで効率が保たれる。
- **専門的な深さを、他のすべてのコストなしに存在させられる。** スキルは必要なだけ詳細で具体的なもの（長いチェックリスト、専門用語、エッジケース）にできる——その詳細さが無関係な会話を肥大化させることなく。
- **能力の拡大に伴ってよりよくスケールする。** 100個目の専門スキルを追加しても、その日最初の会話が遅くなることはない。ほとんどのスキルは必要にならない限りロードされないからだ。

## 自分なりに応用するには

AIアシスタントのための「Xのやり方」という指示が、大きくて増え続ける集合になっていることに気づいたら、一つの肥大化し続けるマスタープロンプトにするのではなく、別々のよくラベル付けされたパッケージに分割することを検討してみてほしい——そして、それぞれの種類のタスクが実際に出てきたときにだけロードする。
