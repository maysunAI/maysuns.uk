# LLM（大規模言語モデル）とは何か

## それが何であるか

大規模言語モデルとは、膨大な量のテキストで訓練され、単語の並びの中で次に何が来るかを予測するニューラルネットワークだ。単純に聞こえるが、これを大規模に——実際の文章、コード、会話の何十億もの例にわたって——うまくやることが、次単語予測というコアの仕組みそのものは変わらないにもかかわらず、文章を書き、問題を推論し、質問に答え、会話を続けることを可能にしている。

## 実際にどう機能しているか、大まかに言うと

- **単語ではなくトークン。** テキストはトークンと呼ばれる断片（多くの場合、単語の一部）に分割される。モデルは文字ではなく数字を扱う——各トークンは数値に対応し、モデルは最も可能性の高い次のトークンを一つずつ予測する。
- **訓練（training）。** モデルは膨大な量のテキストを読み、実際のテキストで次に来るものに予測が近づくように、何十億もの内部パラメータを調整する。これは事前に一度だけ行われる——だからこそすべてのモデルには「知識のカットオフ日」がある。
- **推論（inference）。** 実際にモデルとチャットしているとき、モデルはもう学習していない——ただその訓練済みの予測プロセスを、現在の会話にあるすべて（「コンテキストウィンドウ」）を使って何が来そうかを判断しながら、一トークンずつ実行しているだけだ。

## なぜ「ただの」次単語予測でここまでできるのか

テキストをうまく予測するには、実は文法、事実、推論パターン、コーディングの慣習、トーンといった多くのことを暗黙のうちに捉える必要があることがわかっている。それらは何一つ手作業でプログラムされたわけではない——巨大で多様なデータセット全体にわたってテキストを正確に予測するよう訓練された結果として、自然に現れてきたものだ。

## 知っておくべき重要な限界

- **知識のカットオフ。** モデルは、ある一定の日付までの訓練データにあったことしか「知らない」——ウェブ検索のようなツールや会話に貼り付けられたテキストを通じて直接与えられない限り、それ以降のことは何も知らない。
- **モデルは検証済みの真実ではなく、もっともらしいテキストを予測している。** 自信たっぷりで流暢な答えは、必ずしも正しい答えとは限らない——これが「ハルシネーション（幻覚）」の根本原因であり、モデルは真実と同じ流暢さで、誤ったことを述べることがある。
- **コンテキストウィンドウは有限だ。** モデルが一度に考慮できるテキスト（会話履歴、文書、コード）の量には限りがある。そのウィンドウの外にあるものは、そのターンではモデルには見えない。
