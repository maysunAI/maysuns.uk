# AIセッションをまたぐ永続的メモリ

## 解決する課題

ほとんどのAIアシスタントとの会話はゼロから始まる。新しいセッションのたびに、自分の好み、自分の慣習、前回伝えたはずの繰り返しのフィードバック（「いや、日付はいつもこの形式にしたい」「Xはやらないで、もう3回言った」）を説明し直すことになる。何十回ものセッションを重ねると、これはかなりの量の繰り返しのコンテキストになり、小さいけれど重要な好みがセッション間で失われたり、矛盾したりしやすくなる。

## アイデア

セッションをまたいで持ち越す価値のある一握りの事実を運ぶ、小さく永続的なメモリファイル——意図的に過去のすべての会話の完全な記録*ではない*——を用意する。

- 継続的な好み（「Xをする前は必ず確認して」）
- 繰り返しの修正（「このバグをもう3回説明した、二度と説明しなくていいように書き留めておいて」）
- 作業コンテキストに関する安定した事実（命名規則、物事がどこにあるか、誰が関わっているか）

肝心な設計上の選択は**取捨選択（selectivity）**だ。すべてを記憶しようとするメモリファイルは、メモリが全くないのと同じくらい使いものにならなくなる——参照するには大きすぎ、永続的な事実と一過性のノイズが混ざってしまう。良いメモリファイルは短いままで、生のチャットログではなく小さな「チームwiki」ページを維持するのと同じように、意図的に編集される。

## なぜ重要なのか

- **繰り返しが減る。** 継続的な好みを毎セッション言い直す必要がない。
- **一貫性。** AIの振る舞いは、セッションごとにブレるのではなく、すでに伝えたことと整合したままになる。
- **検査可能。** 不透明な内部状態ではなく普通のファイルであるため、何が「記憶」されているのかを正確に読むことができ、間違って記録されたものは訂正でき、古くなったものは削除できる。

## 自分なりに応用するには

アシスタントがセッションの開始時に読む（あるいは何らかの形でアクセスできる）短いMarkdownファイルを維持する。長期的に覚えておく価値があるものだけを、毎回の会話で自動的にではなく、意図的に自分の言葉で追加する。時間とともに散らかっていく参照文書を整理するのと同じように、時々見直して不要なものを削る。
