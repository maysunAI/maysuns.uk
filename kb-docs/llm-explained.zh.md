# LLM（大语言模型），解释一下

## 它是什么

大语言模型是一种神经网络，通过在海量文本上训练，来预测一串词语中接下来会出现什么。这听起来很简单，但要在规模上把它做好——横跨数十亿条真实写作、代码和对话的样本——正是这种能力让它能够写作、推理解决问题、回答问题、进行对话，尽管从本质上讲，它做的仍然是「预测下一个词」。

## 它到底是如何运作的（高层概览）

- **是词元（token），不是单词。** 文本被切分成称为词元（token）的小块（通常是单词的一部分）。模型处理的是数字，不是字母——每个词元对应一个数字，模型每次预测一个最可能出现的下一个词元。
- **训练（training）。** 模型阅读海量文本，并调整数十亿个内部参数，使其预测结果越来越接近真实文本中实际出现的下一个内容。这个过程是提前进行、只做一次的——这就是为什么每个模型都有一个「知识截止日期（knowledge cutoff）」。
- **推理（inference）。** 当你实际和它聊天时，模型已经不再学习了——它只是在运行那个训练好的预测过程，一次一个词元，利用当前对话中的所有内容（「上下文窗口」，context window）来判断接下来最可能出现什么。

## 为什么「仅仅」预测下一个词就能做到这么多

事实证明，要把文本预测做好，模型必须隐式地掌握大量东西：语法、事实、推理模式、编程惯例、语气风格。这些都不是被人手工编程进去的——而是在被训练去准确预测一个庞大而多样的数据集中的文本时，自然涌现出来的。

## 值得了解的重要局限

- **知识截止日期。** 模型只「知道」截止到某个日期为止训练数据中包含的内容——除非直接把信息提供给它（比如通过网页搜索这样的工具，或者粘贴进对话里的文本），否则它对这之后发生的任何事情都一无所知。
- **它预测的是听起来合理的文本，而不是经过核实的真相。** 一个听起来自信、流畅的答案，并不等同于一个正确的答案——这正是「幻觉（hallucination）」的根本原因，模型会用和陈述真相同样流畅的语气,陈述一些不真实的内容。
- **上下文窗口是有限的。** 模型一次能考虑的文本量（对话历史、文档、代码）是有上限的；超出这个窗口的内容，在当前这一轮里模型是看不到的。
