# Node.jsの基本

## それが何であるか

Node.jsは、JavaScriptコードをウェブブラウザの外で実行できるようにするJavaScriptランタイムだ——サーバー上、コマンドラインツール、プログラムを動かす必要のあるどこでも。Node.js以前、JavaScriptは事実上ブラウザ専用だった。Node.jsは同じ言語を、サーバー、スクリプト、バックエンドアプリケーションを書くための汎用ツールにした。

これはChromeのV8エンジン（Chromeブラウザを動かしているのと同じJavaScriptエンジン）の上に構築されており、**ノンブロッキングでイベント駆動のI/O**を中心に設計されている——つまり、Node.jsのプログラムは、遅い処理（ネットワークリクエスト、ファイルの読み込み、データベースのクエリ）が終わるのをじっと待って何もせずにいることはない。結果が準備できたときのためのコールバックを登録し、その間は他の作業を処理し続ける。

## ノンブロッキングモデルが重要な理由

実際のサーバー作業の多くは*待つこと*だ——データベースの応答を待つ、別のAPIの回答を待つ、ファイルの読み込みが終わるのを待つ。従来のブロッキングモデルは、待機中のタスクごとに新しいスレッドを立ち上げることでこれに対処するが、規模が大きくなると高コストになる。Nodeのイベント駆動モデルは、多数の同時進行の待機処理を単一スレッドで効率的に処理できる。これが、Nodeが API、リアルタイムアプリケーション（チャット、ライブ更新）、I/Oの多いバックエンドサービスで人気の選択肢になった理由の一つだ。

## 何に使われるか

- **APIとバックエンドサーバー** — Node.js（多くの場合Expressのようなフレームワークと組み合わせて）は、JavaScriptでJSON APIやウェブバックエンドを構築するもっとも一般的な方法の一つだ。
- **コマンドラインツール** — 多くの開発者向けツール（ビルドシステム、リンター、パッケージマネージャー）自体がNode.jsで書かれている。
- **フルスタックJavaScript** — フロントエンド（例えばVue、React）とバックエンド（Node.js）の両方で同じ言語を使うことで、コンテキストの切り替えが減り、一部のコード（バリデーションロジック、データ型）を両者間で共有できる。

## エコシステム：npm

Node.jsには**npm**（Node Package Manager）が同梱されており、再利用可能なパッケージの巨大な公開レジストリにアクセスできる——つまり、よくある問題（日付のパース、HTTPリクエストの送信、ウェブサーバーの構築）の多くには、ゼロから解決する必要のない、実績のあるライブラリがすでに存在している。
