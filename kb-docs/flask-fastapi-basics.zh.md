# Flask 与 FastAPI 基础

## 它们是什么

Flask和FastAPI都是轻量级的Python Web框架，用于构建API和Web后端。两者都能让你定义路由（URL）以及请求到达时该发生什么，而不需要承担一个庞大的、大而全的全栈框架的开销——但它们建立在不同的设计理念之上。

## Flask

Flask刻意保持极简，且不强加太多约定（unopinionated）。它提供一个很小的核心——路由、请求/响应处理——其余几乎所有东西（数据库访问、身份验证、数据校验）都留给你，按需通过扩展或自己的代码去添加。这使得Flask既灵活又容易理解：幕后没有太多「魔法」在发生，这也是它在小型服务、原型开发,以及那些你想要完全掌控整个技术栈的项目中广受欢迎的部分原因。

## FastAPI

FastAPI是一个更新的框架，围绕几个具体理念构建：

- **以Python类型提示（type hints）作为唯一的事实来源。** 你使用标准的Python类型注解来声明请求/响应数据应该是什么形状，FastAPI利用这些声明自动校验传入的数据，并据此生成文档。
- **自动生成的交互式API文档。** FastAPI直接从你的代码生成一个可浏览、可交互的API文档页面（通过OpenAPI标准），不需要额外单独编写文档这一步。
- **从一开始就内置异步支持。** FastAPI围绕Python的 `async`/`await` 语法设计，天然适合那些需要高效处理大量并发I/O密集型工作（等待数据库、等待外部API）的API。

## 该如何选择

- **Flask**适合你想要最大程度的简单和掌控力，或者你在构建一些不需要FastAPI那种额外结构的小东西时。
- **FastAPI**适合你在构建一个「正经的」API，能从自动校验、自动生成文档以及开箱即用的异步性能中获益——这也是为什么它已经成为新Python API项目的常见默认选择，尤其是那些需要被其他服务调用、或者需要面向想要清晰API文档的前端团队的项目。

两者都远比大型全栈框架轻量得多，而且两者之所以常被选用，*正是因为*它们只让你添加项目实际需要的那部分组件。
