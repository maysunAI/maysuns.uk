# 跨AI会话的持久化记忆

## 解决的问题

大多数AI助手对话都是从零开始的。每次新会话，你都要重新解释自己的偏好、自己的约定、上次已经给过的重复反馈（「不，我一直想要这种日期格式」「别做X，我已经说了三次了」）。经过几十次会话之后，这累积成了大量重复的上下文——一些虽小但重要的偏好，很容易在不同会话之间丢失或产生矛盾。

## 核心思路

一个小型的、持久的记忆文件——刻意地*不是*过往每次对话的完整记录——用来把少数值得跨会话保留的事实带下去：

- 一贯的偏好（「做X之前总是先确认」）
- 反复出现的纠正（「这个bug我已经解释过三次了，写下来免得再解释一次」）
- 关于工作情境的稳定事实（命名约定、东西放在哪里、涉及哪些人）

关键的设计选择在于**有选择性（selectivity）**。一份试图记录一切的记忆文件，会变得和完全没有记忆一样没用——它太大了，无法有效参考，而且把持久性的事实和一次性的噪音混在了一起。好的记忆文件保持简短，并且被有意识地编辑，就像维护一个小型「团队wiki」页面，而不是一份原始的聊天记录。

## 为什么重要

- **减少重复。** 一贯的偏好不需要每次会话都重新说一遍。
- **一致性。** AI的行为会持续与你已经告诉过它的内容保持一致，而不是在不同会话之间逐渐漂移。
- **可检查。** 因为它是一份普通文件而不是不透明的内部状态，你可以准确读到「被记住」的是什么，如果记录出错可以纠正,过时的内容也可以删除。

## 如何自己应用这套方法

维护一份简短的markdown文件，让助手在会话开始时读取（或以其他方式获得访问权限）。当某件事值得长期记住时，用你自己的话有意识地把它加进去——而不是每次交流都自动添加。像整理任何随时间累积杂物的参考文档一样，偶尔回顾并做一些精简。
