# 自定义斜杠命令

## 解决的问题

给AI助手的一些指示并不是一次性的——它们是你反复使用的、可重复的多步骤流程：「按照我们的清单审查这段代码」「估算一下这个任务有多大」「把这个项目打包准备发布」。每次都把同样的多段落指示重新打一遍很麻烦，更糟的是，这会带来漂移——你每次的措辞都略有不同，得到的结果也略有不同。

## 核心思路

斜杠命令把一个可重复的工作流打包成一个可复用的触发器——通常是以 `/` 开头的简短名字，比如 `/estimate` 或 `/publish`。这一个词的背后，藏着完整的一整套步骤：要检查什么、按什么顺序做、要产出什么格式的输出、如果出了问题该标注什么。

一旦定义好，调用这个工作流就从重新打一段小作文变成了一行命令——同样重要的是，它是*一致的*。同一个命令每次都会产生同一种流程，这对任何你反复依赖的东西（比如一份你不想不小心漏掉某一步的发布前检查清单）来说都很重要。

## 为什么重要

- **一致性胜过重复劳动。** 一个你经常使用的工作流，每次的表现应该是一样的，而不是取决于你当天恰好把指示表述得有多好。
- **降低养成好习惯的摩擦。** 如果运行一份规范的检查清单只需要一个词而不是五分钟的打字，你实际执行它的可能性就更高。
- **易于发现、易于分享。** 一套命名好的命令库，比「我平时是这么问的」这种零散的记忆，更容易交给别人（或者未来的自己）。

## 如何自己应用这套方法

留意那些你发现自己以大致相同的形式打了两次以上的指示——这些就是命名命令的候选对象。把步骤写下来一次，给它起一个简短好记的名字，然后重复使用它，而不是每次都从头重新推导一遍指示。
