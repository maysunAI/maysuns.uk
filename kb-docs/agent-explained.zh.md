# AI智能体（Agent），解释一下

## 解决的问题

一个普通的聊天机器人一次只回答一个问题：你问，它答，对话就此结束。但许多现实任务并不是一问一答就能解决的——「修好这个bug」「研究这个主题并写一份摘要」「重新整理这些文件」——这些任务需要多个步骤，需要根据刚发生的事情来决定下一步该做什么，并且需要真正能够采取行动（而不只是描述别人应该做什么）。

## 它是什么

AI智能体（agent）是一种能够规划一系列步骤、使用工具采取真实行动（读写文件、运行命令、调用API、浏览网页）、观察这些行动的结果，并决定下一步该做什么的模型——如此循环往复，直到任务完成，且大多数情况下不需要人类逐步批准每一个动作。

其核心循环通常是：**观察 → 决策 → 行动 → 观察结果 → 再次决策。** 智能体和普通聊天机器人的区别就在最后这一环：它不只是产出一个答案，而是真的采取行动、查看发生了什么，并据此调整。

## 什么才算「智能体式（agentic）」

不是每一个AI功能都是智能体。一个有用的判断标准是：它是只产出一段文本让人类去执行，还是它自己在多个步骤中做决策并采取行动？

- **不是智能体式的**：「总结这份文档」——一个输入，一个输出，没有持续的决策过程。
- **是智能体式的**：「让这套测试套件全部通过」——模型必须运行测试、阅读失败信息、决定要改什么、编辑代码、再次运行测试，如此反复直到完成。

## 为什么重要

- **可以委托出去的事情更多了。** 那些过去每个决策点都需要人类在场的多步骤、开放式任务，现在可以直接交出去，人类只需要审查结果，而不必对每一步都进行微观管理。
- **工具的使用才是让它「变得真实」的关键。** 一个智能体的能力上限就是它能调用的工具的上限——文件访问、代码执行、搜索、浏览器控制。推理能力固然重要，但如果没有真正的工具，那终究还是纸上谈兵。
- **自主性需要护栏。** 因为智能体不经过每一步的批准就采取行动，允许它在无人监督情况下做多少事情，就成了一个真正的安全问题——具体在实践中如何管理，可以参考权限/信任级别相关的约定。
