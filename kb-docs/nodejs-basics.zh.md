# Node.js 基础

## 它是什么

Node.js是一个JavaScript运行时，它让JavaScript代码可以在网页浏览器之外运行——在服务器上、在命令行工具里、在任何需要运行程序的地方。在Node.js出现之前，JavaScript实际上只能在浏览器里使用；Node.js把同一种语言变成了一个可以用来编写服务器、脚本和后端应用的通用工具。

它构建在Chrome的V8引擎之上（也就是驱动Chrome浏览器的那个JavaScript引擎），其设计围绕着**非阻塞、事件驱动的I/O**——也就是说，Node.js程序不会因为一个较慢的操作（一次网络请求、一次文件读取、一次数据库查询）而闲着不动、干等它完成；它会为「结果就绪」这件事注册一个回调,同时继续处理其他工作。

## 为什么非阻塞模型很重要

现实世界中服务器的大量工作其实是在*等待*——等待数据库响应，等待另一个API回复，等待文件读取完成。传统的阻塞模型通过为每一个等待中的任务开启一个新线程来处理这个问题，但这在规模扩大后代价高昂。Node的事件驱动模型能在单个线程上高效处理大量并发的等待操作，这也是Node成为构建API、实时应用（聊天、实时更新）以及I/O密集型后端服务的热门选择的原因之一。

## 它通常被用来做什么

- **API和后端服务器** ——Node.js（通常搭配像Express这样的框架）是用JavaScript构建JSON API或Web后端最常见的方式之一。
- **命令行工具** ——许多开发者工具本身（构建系统、代码检查工具、包管理器）就是用Node.js编写的。
- **全栈JavaScript** ——在前端（比如Vue、React）和后端（Node.js）使用同一种语言，意味着更少的上下文切换，并且能让一些代码（校验逻辑、数据类型）在两端之间共享。

## 生态系统：npm

Node.js自带**npm**（Node Package Manager，Node包管理器），可以访问一个庞大的公共可复用包注册表——这意味着大多数常见问题（解析日期、发起HTTP请求、构建一个Web服务器）都已经有了经过充分测试的现成库,而不需要从零开始解决。
